# Quick Task: Optimize audio callback hot path

## Context

CoreAudio logs show `ClientHALIODurationExceededBudget` overloads on the Arturia AF16Rig — the HAL IO callback takes 14.6ms against an 11.3ms budget (512 samples @ 48kHz). Two hotspots in the recording callback: heap allocation per buffer and redundant RMS iteration.

## Tasks

### Task 1: Pre-allocate channel remap buffer
- **files:** `crates/wallflower-core/src/recording/mod.rs`
- **action:** Move `Vec::with_capacity()` outside the closure. Reuse via `clear()` each callback.
- **verify:** `cargo build --release -p wallflower-core` succeeds
- **done:** No heap allocation per audio callback

### Task 2: Eliminate double RMS computation
- **files:** `crates/wallflower-core/src/recording/mod.rs`, `crates/wallflower-core/src/recording/silence.rs`
- **action:** Add `process_with_rms()` to SilenceDetector. Compute RMS once in callback for level metering, pass to silence detector.
- **verify:** `cargo test -p wallflower-core -- recording` all pass
- **done:** Buffer iterated once instead of twice per callback
