   - Accept `app: tauri::AppHandle`, `state: tauri::State<'_, AppState>`, `id: String`
   - Lock the DB, call `db::delete_jam(&db.conn, &id)` to delete the record and get the JamRecord back
   - After DB deletion succeeds, clean up files from disk (best-effort, log warnings on failure, do not fail the command):
     a. Delete audio file: resolve `config.storage_dir.join(&jam.filename)`, remove if exists

     c. Delete photos: glob `dirs::data_dir().join("wallflower/photos/{id}_*")` -- iterate directory entries matching prefix `{id}_`, remove each

Since bookmarks are already cascade-deleted from the DB, we can instead glob the stem_cache directory. However, bookmark IDs are not available after cascade. Instead, before the DB delete, query bookmarks for this jam_id to collect their IDs. Then after DB delete, remove `stem_cache/{bookmark_id}/` for each.
   - Adjust the function to: (a) lock db, (b) query bookmark IDs for the jam via `db::list_bookmarks(conn, &id)` (already exists in the bookmarks module), (c) call `db::delete_jam`, (d) unlock db, (e) clean up files.
   - Import needed: `use wallflower_core::db::bookmarks;` (check if the bookmarks module has `list_bookmarks` -- it should since `get_bookmarks` Tauri command exists). If the function is named differently, use the actual name from the codebase.
