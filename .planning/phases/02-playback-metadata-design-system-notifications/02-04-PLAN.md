  <action>
    **Patches folder watcher** (crates/wallflower-app/src/lib.rs):
    Add a background task that watches ~/wallflower/patches/ for new image files using the `notify` crate (already in wallflower-core dependencies).

    Implementation:
    1. In the Tauri setup hook (`.setup(|app| { ... })`), spawn a tokio task that:
       - Creates the ~/wallflower/patches/ directory if it doesn't exist
       - Uses `notify::RecommendedWatcher` to watch the directory for Create events
       - On new file with image extension (.jpg, .jpeg, .png, .gif, .webp, .heic):
         a. Get the most recent jam from DB via `get_most_recent_jam()` (returns first result ordered by imported_at DESC)
         b. If a jam exists: call `photos::store_photo()` to copy to app data dir, then `db::insert_photo()` with source="patches_folder"
         c. Send native notification: title "Patch Photo Attached", body "New photo linked to {jam_name}"
         d. Emit Tauri event `photo-auto-attached` with { jamId, jamName } so frontend can refresh
    2. Handle the watcher in a separate thread (notify watcher is synchronous, use crossbeam or mpsc channel to bridge to tokio)

    **DB helper** (crates/wallflower-core/src/db/mod.rs):
    Add function `get_most_recent_jam(conn: &Connection) -> Result<Option<JamRecord>>` that returns the first jam ordered by imported_at DESC LIMIT 1. This is used by the patches watcher to determine which jam to auto-attach to.

    **Notification triggers** (crates/wallflower-app/src/lib.rs):
    Add notification calls at these points:
    1. **Import complete** -- after successful file import (in the import_files command or wherever import logic lives):
       - Single file: title "Import Complete", body "{filename} added to your library"
       - Multiple files: title "Import Complete", body "{count} files added to your library"
    2. **Patch photo auto-attached** -- already handled in patches watcher above
    3. **Device connected** -- placeholder: title "Recorder Connected", body "{device name} detected. Open Wallflower to import." (Wire when device detection is implemented in Phase 1 remaining plans)

    Use `tauri_plugin_notification::NotificationExt` for sending:
    ```rust
    use tauri_plugin_notification::NotificationExt;

    fn notify(app: &tauri::AppHandle, title: &str, body: &str) {
        let _ = app.notification()
            .builder()
            .title(title)
            .body(body)
            .show();
    }
    ```

    **Frontend event listener** (src/app/layout.tsx or a hook):
    Listen for `photo-auto-attached` Tauri event to show an in-app toast:
    ```typescript
    import { listen } from '@tauri-apps/api/event';
    import { toast } from 'sonner';

    // In layout or a global hook:
    useEffect(() => {
      const unlisten = listen('photo-auto-attached', (event) => {
        const { jamName } = event.payload as { jamId: string; jamName: string };
        toast('Patch photo auto-attached to ' + jamName, { duration: 4000 });
      });
      return () => { unlisten.then(fn => fn()); };
    }, []);
    ```
