# Phase 2 — UI Design Contract

> Visual and interaction contract for the Wallflower design language, waveform playback, timeline browser, metadata editing, and native notifications. This phase transforms Phase 1's functional shell into the signature Mutable Instruments-inspired creative tool experience. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (customized preset, overriding Phase 1 defaults) |
| Preset | custom — dark base, warm accent tokens, extended border-radius |
| Component library | radix (via shadcn) + custom Tailwind components |
| Icon library | lucide-react |
| Font | Plus Jakarta Sans (geometric sans-serif, per D-03) |

**Phase 2 transition:** Phase 1 used system fonts and light-mode shadcn defaults. Phase 2 replaces the color tokens, typography, and border-radius tokens to establish the Wallflower design language. All Phase 1 components must be restyled to match.

**Font loading:** Plus Jakarta Sans loaded via `@fontsource/plus-jakarta-sans` npm package (self-hosted, no external network requests). Weights 400 and 600 only.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding, badge internal padding |
| sm | 8px | Compact element spacing, tag chip gaps, transport bar icon gaps |
| md | 16px | Default element spacing, form field gaps, card internal padding |
| lg | 24px | Section padding, metadata group spacing, waveform-to-metadata gap |
| xl | 32px | Layout gaps between major sections, tab bar padding |
| 2xl | 48px | Page margins (left/right), major section breaks |
| 3xl | 64px | Page-level vertical spacing (content area top padding) |

Exceptions:
- Transport bar height: 56px (not a spacing token, a fixed component dimension — accommodates play button touch target of 44px with 6px vertical padding)
- Waveform overview bar height: 48px (fixed component dimension)
- Waveform detail view height: 200px (fixed component dimension, resizable in future phases)
- Jam card mini-waveform thumbnail height: 40px (fixed component dimension)

| Role | Size | Weight | Line Height | Usage in Phase 2 |
|------|------|--------|-------------|-------------------|
| Body | 14px | 400 (regular) | 1.5 | Metadata values, notes text, settings descriptions, toast messages |

### Accent Reserved For (explicit list)

1. Play/pause button in transport bar (filled accent background)
2. Waveform fill color (the primary visual element)
3. Active tab indicator (2px underline)
4. Primary CTA buttons ("Import Files", "Save Export")
5. Playback position indicator (vertical line on waveform)
6. Tag chip addition button (+ icon)
7. Focus ring on interactive elements
8. Progress indicators (import progress, notification linking)

**Accent rationale:** Amber/coral (hsl 28 90% 58%) is warm, organic, and inspired by Mutable Instruments' teal-to-warm palette. It reads clearly against the dark background and creates a strong focal point on the waveform — the most important visual element in Phase 2.

### Light Theme (deferred, tokens defined for forward compatibility)

Light theme is NOT implemented in Phase 2. These tokens exist only to ensure the CSS variable architecture supports a future toggle.

## Focal Point

**Primary visual anchor:** The waveform detail view, rendered in the accent color against the dark secondary surface. When a user opens a jam, the waveform is the largest, most colorful element on the page — it immediately communicates "this is about the music."

**Secondary anchor:** The persistent transport bar at the bottom of the screen. The play/pause button uses an accent-filled circle, always visible regardless of scroll position.

## Layout Contract

### App Shell (updated from Phase 1)

```
+--------------------------------------------------+
| [Wallflower]              (native title bar)      |
+--------------------------------------------------+
| [ Library ]  [ Settings ]        (top tab bar)    |
+--------------------------------------------------+
|                                                    |
|  (active tab content area — scrollable)            |
|                                                    |
+--------------------------------------------------+
| [<<] [|>] [>>]  0:00 / 1:23:45   (transport bar) |
+--------------------------------------------------+
```

- Title bar: Native Tauri window chrome (macOS traffic lights). No custom title bar.
- Tab bar: Full width, secondary background (#1D2129), xl (32px) horizontal padding, md (16px) vertical padding. Active tab uses accent underline (2px solid #E8863A).
- Content area: 2xl (48px) horizontal padding, lg (24px) top padding. Scrollable. Bottom padding of 56px to clear transport bar.
- Transport bar: Fixed to bottom. Full width. 56px height. Secondary background. Always visible. Appears after any jam has been played (hidden on fresh install with empty library).

### Library Tab — Timeline Browser

```
+--------------------------------------------------+
|  Library                           [Import Files] |
+--------------------------------------------------+
|                                                    |
|  Today                                             |
|  +----------------------------------------------+ |
|  | [waveform] jam-2026-04-19  1:23:45  WAV      | |
|  |            #eurorack #ambient  @alice         | |
|  +----------------------------------------------+ |
|  +----------------------------------------------+ |
|  | [waveform] session-evening   0:45:12  FLAC   | |
|  |            #modular          @bob             | |
|  +----------------------------------------------+ |
|                                                    |
|  Yesterday                                         |
|  +----------------------------------------------+ |
|  | [waveform] long-jam-04-18   2:01:33  WAV     | |
|  |            (no tags)                          | |
|  +----------------------------------------------+ |
|                                                    |
+--------------------------------------------------+
```

- Date group headers: Label size (12px), muted foreground, uppercase, lg (24px) top margin above each group, sm (8px) bottom margin.
- Date grouping logic: "Today", "Yesterday", then exact date (e.g., "April 17, 2026") for the past 7 days, then "Week of April 7" for the past month, then "March 2026" month headers beyond that.
- Jam cards: Secondary background, 12px border-radius (rounded-xl — organic Mutable Instruments feel), md (16px) internal padding, sm (8px) gap between cards.
- Card layout: Mini-waveform thumbnail (40px height, full card width) at top. Below: jam name (body/14px/semibold) + duration + format badge on one line, tags + collaborators on second line (label/12px, muted foreground). Key + BPM placeholders shown as empty badges (filled by Phase 4 analysis).
- Click card: Navigates to jam detail view.

### Jam Detail View (drill-down from Library)

```
+--------------------------------------------------+
|  [<- Library]   jam-2026-04-19                    |
+--------------------------------------------------+
|                                                    |
|  [=========== overview waveform ===========] 48px  |
|                                                    |
|  [====== detail waveform (zoomable) ======] 200px  |
|                                                    |
|  --- Metadata ---                                  |
|                                                    |
|  Tags      [eurorack] [ambient] [+]               |
|  Collab    [@alice] [@bob] [+]                     |
|  Gear      [Prophet 6] [Moog DFAM] [+]            |
|  Location  [Home Studio              ]             |
|  Date      April 19, 2026 at 3:45 PM              |
|                                                    |
|  Notes                                             |
|  [Free-form text area for session notes...    ]    |
|                                                    |
|  Patch Notes                                       |
|  [Text area for synth/eurorack settings...    ]    |
|                                                    |
|  Photos                                            |
|  [img] [img] [img]  [+ Drop photos here]          |
|                                                    |
+--------------------------------------------------+
```

- Back navigation: Text link "< Library" at top-left, body size, accent color. Click returns to timeline at previous scroll position.
- Overview waveform: Full width, 48px height, secondary background. Shows entire recording. Click to jump to position. Highlighted region shows what the detail view is displaying.
- Detail waveform: Full width, 200px height, secondary background. Zoomable via scroll wheel or pinch. Shows waveform in accent color (#E8863A), played region in darker variant (#B55E20). Playback position as 2px accent vertical line.
- Metadata section: lg (24px) top margin below waveform. Each metadata row has md (16px) vertical spacing.
- Tag/collaborator/instrument chips: 24px height, sm (8px) horizontal padding, xs (4px) gap between chips, 12px border-radius (rounded-xl). Background: surface elevated (#272C36). Text: foreground (#E2E4E8). "+" button: accent border, accent text.
- Autocomplete popover: Appears below the "+" chip when clicked. Shows previously-used values filtered by typing. Popover uses surface elevated background, foreground text.
- Notes and patch notes: Textarea, 4 visible lines (expandable), body size (14px), secondary background, border color border. Auto-saves on blur or after 1 second of inactivity.
- Photo gallery: Grid layout, 3 columns, sm (8px) gap. Each photo: 1:1 aspect ratio thumbnail, 8px border-radius. Click to view full size in a lightbox overlay. Drop zone: dashed border (border color), "Drop photos here" text (muted foreground). Drag-over: accent border.
- Live save: All metadata changes save immediately (META-09). No save button. A subtle "Saved" indicator (muted foreground, fades after 2 seconds) appears near the changed field.

## Interaction States

### Waveform Navigation
| State | Visual |
|-------|--------|
| Default | Overview shows full waveform in accent color. Detail shows zoomed region. |
| Hover on overview | Cursor changes to crosshair. Tooltip shows timecode at cursor position. |
| Click overview | Detail view jumps to center on clicked position. Smooth scroll animation (200ms ease-out). |
| Scroll on detail | Zoom in/out. Minimum zoom: full recording fits in view. Maximum zoom: ~10 seconds visible. |
| Playing | Playback position line (2px accent) moves left-to-right in detail view. Detail view auto-scrolls to follow playhead. Played region uses waveform-played color. |
| Scrub | Click-drag on detail waveform seeks to that position. Audio plays from new position immediately. |

### Transport Bar
| State | Visual |
|-------|--------|
| No jam loaded | Transport bar hidden |
| Jam loaded, paused | Play button (accent filled circle, 44px), skip-back/skip-forward (ghost buttons, 32px). Time display: "0:00 / 1:23:45" in label size. Jam name truncated with ellipsis. |
| Playing | Play button becomes pause button (same accent circle). Time display updates in real time. |
| Hover transport | Button hover states use surface elevated background |

### Tag/Chip Interaction
| State | Visual |
|-------|--------|
| Default chip | Surface elevated background, foreground text, 12px border-radius |
| Hover chip | Border becomes accent color (subtle highlight) |
| Click chip | No action (tags are display-only; remove via X icon) |
| Remove chip | X icon appears on hover. Click X opens inline confirmation (chip background flashes destructive color briefly, then removes). |
| Add chip (+) | Click "+" opens inline text input with autocomplete popover below |
| Autocomplete | Popover lists matching previously-used values. Arrow keys to navigate, Enter to select, Escape to dismiss. First item pre-selected. |

### Photo Attachment
| State | Visual |
|-------|--------|
| Default gallery | Grid of thumbnails + dashed drop zone |
| Drag over | Drop zone border becomes accent, background gains 10% accent tint, text changes to "Release to attach" |
| Photo added | Thumbnail appears with a brief scale-up animation (150ms). Toast: "Photo attached to {jam name}" |
| Auto-attach (from patches folder) | Toast: "Patch photo auto-attached to {jam name}" with thumbnail preview in toast |
| Remove photo | Hover thumbnail shows X overlay. Click X shows confirmation dialog: "Remove this photo from {jam name}?" |

### Metadata Live-Save
| State | Visual |
|-------|--------|
| Editing | Field has accent border (focused state) |
| Saving | No spinner. Field border returns to default. Subtle "Saved" text fades in near the field (muted foreground, 12px) then fades out after 2 seconds. |
| Error | Field border becomes destructive. Toast: "Could not save changes. Check your connection and try again." |

## Phase 2 Specific Notes

1. **Design language establishment.** This phase defines the Wallflower visual identity. All Phase 1 components must be restyled to match the new dark theme and warm accent palette. The shadcn CSS variables are the mechanism — update the CSS variables, and all shadcn components inherit the new look.

2. **Dark mode is the only mode.** Light theme tokens are defined for forward compatibility but are NOT implemented. No theme toggle in Phase 2.

3. **Waveform is the hero.** The waveform visualization in accent color is the signature visual element. It should feel alive, warm, and musical — not clinical. wavesurfer.js handles rendering; the design contract specifies colors and dimensions, not implementation.

4. **Pre-computed peaks are a backend dependency.** PLAY-01 requires multi-resolution peaks served from the Rust backend. The waveform components depend on this API. If peaks are not ready, show a loading skeleton (pulsing secondary background bar at the waveform dimensions).

5. **Accessibility baseline continues.** All interactive elements must have visible focus rings (2px accent outline, 2px offset). All form inputs must have associated labels. Tag chips must be keyboard-navigable (arrow keys between chips, Delete/Backspace to remove, Enter on "+" to add). Full accessibility audit remains in Phase 6.

6. **Border radius convention.** Mutable Instruments aesthetic uses generous rounding. Cards and chips use 12px (rounded-xl). Buttons use 8px (rounded-lg). Input fields use 8px (rounded-lg). This creates a consistently organic, non-corporate feel.

7. **No pagination needed yet.** Timeline browser uses virtual scrolling only if performance degrades. A musician's library in early use will be manageable. Add virtual scrolling as a performance optimization, not a Phase 2 requirement.
