# Phase 4 — UI Design Contract

> Visual and interaction contract for ML analysis pipeline UI: analysis result badges on jam cards, section markers and loop brackets on waveforms, progressive analysis status, search and filter bar, model management settings, and re-analysis controls. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (customized preset, inherited from Phase 2) |
| Preset | custom -- dark base, warm accent tokens, extended border-radius |
| Component library | radix (via shadcn) + custom Tailwind components |

### Phase 4 Addition: Section Marker Colors

Six distinct section type colors for waveform markers. These are muted pastels that read clearly against the dark waveform background without competing with the accent waveform fill.

| Section Type | Value | Usage |
|-------------|-------|-------|
| Intro/Outro | hsl(210 60% 65%) / #5B9BD5 | Blue -- structural bookends |
| Verse / Part A | hsl(150 50% 55%) / #4CAF7D | Green -- primary musical sections |
| Chorus / Part B | hsl(340 60% 65%) / #D35D8A | Pink -- contrasting musical sections |
| Bridge / Transition | hsl(45 70% 60%) / #D4A843 | Gold -- transitional passages |
| Loop (repeating) | hsl(270 50% 65%) / #9B7DC9 | Purple -- detected loops and repeating regions |
| Unknown / Other | hsl(220 10% 50%) / #747A88 | Muted foreground -- unclassified sections |

**Section marker rendering:** Vertical lines are 2px wide, full waveform height, at 60% opacity. Labels sit above the waveform in the corresponding section color at label size (12px). Colors are chosen to be distinguishable from each other and from the accent amber waveform fill.

### Loop Bracket Colors

Loop brackets use the Loop section color (purple, hsl 270 50% 65%) at 40% opacity for the bracket line, with label text at full opacity. Evolving loops add a small tilde (~) character after the repeat count.

### Analysis Status Colors

| State | Color | Usage |

## Focal Point

**Primary visual anchor:** The waveform detail view, now enriched with colored section markers and loop brackets. Section markers "light up" progressively as analysis completes -- the waveform transforms from a raw audio visualization into a structurally annotated musical map.

**Secondary anchor:** The analysis summary row (compact chips for Key, BPM, Sections, Loops) positioned between the waveform and the metadata editor. This row is the "analysis dashboard" for the jam.

## Layout Contract

### Library Tab -- Timeline with Filter Bar (D-12)

```
+--------------------------------------------------+
|  Library                           [Import Files] |
+--------------------------------------------------+
|                                                    |
|  [Search...    ] [Key v] [110-130 BPM] [Tags v]   |
|  [Collaborators v] [Instruments v] [Date range v]  |
|  Active: [Bb minor x] [110-130 BPM x]  2 results  |
|                                                    |
|  Today                                             |
|  +----------------------------------------------+ |
|  | [waveform] jam-session  Bbm 120  1:23:45 WAV  | |
|  |            #eurorack #ambient  @alice          | |
|  +----------------------------------------------+ |
|                                                    |
+--------------------------------------------------+
```

- Filter bar: Full width, secondary background (#1D2129), md (16px) padding, sm (8px) gap between filter controls. Positioned above the date-grouped timeline, below the page heading. Sticky to top on scroll.
- Search input: 240px width, secondary background, border, placeholder "Search jams...". Magnifying glass icon (lucide `Search`, 14px) as left adornment. Clear button (lucide `X`, 14px) appears when text is present.
- Filter dropdowns: Each uses Select component. Default label shows field name (e.g., "Key", "Tags"). When values selected, label updates to show count (e.g., "Key (2)"). Surface-elevated background on open.
- Active filter chips: Appear in a row below the filter controls. Each chip shows the filter value with an X button to remove. Accent background (#E8863A), primary-foreground text (#FFFFFF), 12px border-radius. sm (8px) gap between chips.
- Results count: Right-aligned in the active filters row. Label size (12px), muted-foreground. Format: "{N} results" or "No results".
- Filters combine with AND logic (D-12). Filtering is instant -- no loading state, queries local SQLite.

### Filter Bar -- Collapsed State (no active filters)

```
+--------------------------------------------------+
|  [Search...    ] [Key v] [BPM v] [Tags v] [More v]|
+--------------------------------------------------+
```

- When no filters are active, the active chips row is hidden.
- "More" dropdown contains: Collaborators, Instruments, Date range. These are less commonly used and reduce initial filter bar width.

### Jam Card -- With Analysis Badges (D-01)

```
+----------------------------------------------+
| [=========== mini waveform ===========]       |
|                                               |
| jam-2026-04-19  Bbm  120  1:23:45  WAV       |
| #eurorack #ambient  @alice                    |
+----------------------------------------------+
```

- Key badge: Label size (12px), semibold (600), foreground color when filled, muted-foreground when pending ("--"). Background: surface-elevated (#272C36). Rounded-xl (12px). Padding: xs (4px) vertical, sm (8px) horizontal.
- BPM badge: Same styling as key badge. Value in tabular-nums. Shows integer only (e.g., "120" not "120.3").
- Pending state: Both badges show "--" in muted-foreground. Layout is reserved -- badges are always present at their full width to prevent layout shift (D-01).
- Analyzing state: A subtle "Analyzing..." text replaces the badges temporarily, with a pulsing accent dot (4px, same animation as recording dot but using accent color). Animation: opacity 1.0 to 0.3 over 2 seconds.

### Section Markers on Waveform (D-02)

Section boundaries rendered as colored vertical lines on both WaveformOverview and WaveformDetail:

- Vertical line: 2px wide, full waveform height, section type color at 60% opacity
- Label (WaveformDetail only): Section type name positioned above the waveform, label size (12px), section type color at full opacity. Labels positioned at the start of each section. If labels would overlap, later labels are omitted (show on hover instead).
- Label format: Short names -- "Intro", "Verse A", "Chorus", "Bridge", "Loop 1", "Outro"
- Hover on section region: Background fills with section color at 8% opacity across the waveform region. Tooltip shows section duration.

### Loop Brackets on Waveform (D-05)

Loop brackets rendered above section markers on WaveformDetail:

```
  |---- Loop A x4 ----|---- Loop A x4 (evolving) ----|
  |  Intro  |  Verse A  |  Bridge  |  Chorus  |  Outro |

```

- Bracket line: 2px solid, loop color (purple) at 40% opacity, horizontal line 12px above the waveform top edge
- Bracket ends: Small vertical ticks (8px tall, 2px wide) at start and end of loop region
- Label: Centered above the bracket, label size (12px), loop color at full opacity. Format: "Loop {letter} x{count}" or "Loop {letter} x{count} (evolving)" for evolving loops (D-05)
- Bracket region: If loop bracket area becomes too narrow for the label, show label on hover only

Small, persistent badges in the top-right corner of the waveform detail view:

- Position: 8px from top-right corner of waveform container
- Badge style: Dominant background (#151921) at 80% opacity, foreground text, label size (12px), 8px border-radius, xs (4px) padding
- Content: "{key} | {bpm} BPM" (e.g., "Bbm | 120 BPM")

### Manual Override Interaction (D-18)

In the jam detail analysis summary row, clicking on a Key or BPM badge opens an inline edit mode:

- Key badge click: Badge transforms into a Select dropdown with all musical keys (C major, C minor, C# major, ... B minor). Current value pre-selected. Escape cancels. Selection confirms.
- BPM badge click: Badge transforms into a compact number input (60px width). Current value pre-filled. Enter confirms. Escape cancels. Valid range: 20-300.
- After manual set: Badge shows the pencil icon (accent color), value persists across re-analysis (D-18, D-19).
- Clear manual override: Small X button appears on hover next to pencil icon. Click clears manual value and allows re-analysis to set it.

## Phase 4 Specific Notes

1. **Analysis results "light up" progressively.** The core UX principle: jam cards start with "--" placeholder badges and fill in as each analysis step completes (D-01, D-09). Section markers appear on the waveform only after the sections step finishes. This progressive disclosure avoids any "waiting for analysis" dead state -- the jam is always usable.

2. **Layout stability is mandatory.** Key and BPM badge slots on jam cards must reserve their full width from initial render. No layout shift when values populate. Use fixed-width containers or min-width constraints on badge slots.

3. **Section markers follow Ableton conventions.** Colored vertical lines with short labels are familiar to DAW users (D-02). The color palette for sections is deliberately distinct from the accent amber to avoid confusion with the waveform fill or playback position indicator.

4. **Filter bar is local-only, instant.** All filtering queries local SQLite. No loading spinners, no debounce on filter changes. Results update on every interaction. The filter bar should feel as responsive as macOS Finder smart folders.

5. **Sidecar is invisible to the user.** The Python sidecar lifecycle (D-06, D-07, D-08) is fully abstracted. Users see "Analyzing..." and "Analysis complete" -- never "Python process" or "sidecar." The only exception is the recovery toast "Restarting analysis engine..." which uses "analysis engine" as the user-facing term.

6. **Model management is informational, not interactive.** Settings shows models and their status but provides no install/uninstall controls (D-11). Models download automatically on first analysis. The UI is a transparency window, not a control panel.

7. **Manual overrides are sacred.** AI-detected values can be manually corrected (D-18). Manual values show a pencil icon and are never overwritten by re-analysis (D-19). Clearing a manual override is explicit (X button on hover). This respects the musician's ear over the algorithm.

8. **Recording priority remains absolute.** When recording starts (D-17), analysis is interrupted and re-queued. The analysis status badge should gracefully transition from "Analyzing..." to the pending state without error messaging. This is expected behavior, not an error.

9. **Queue priority follows attention.** The currently-viewed jam jumps to front of the analysis queue (D-16). This is a backend behavior but the UI should reflect it: when a user opens a jam detail that has pending analysis, the "Analyzing..." status should appear quickly as the jam is prioritized.

10. **Dual-handle tempo slider.** The BPM filter uses a range slider with two handles (D-13). This is not a standard shadcn Slider -- it requires a custom implementation atop the Slider primitive or a dual-thumb variant. The visual contract: 4px track, 16px thumbs, accent fill between handles, border background for unfilled regions. Label below shows "110 - 130 BPM" format.
