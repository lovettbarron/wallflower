**Create `src/components/explore/SamplePreviewPanel.tsx`:**

A persistent bottom panel per D-08, D-09, D-15 that appears when a sample is selected. Height: 220px. Background: #1D2129. Top border: 1px solid border color. Slides up with 200ms ease-out transition.

Props: `{ sample: SampleRecord; onNavigateToJam: (jamId: string) => void; onClose: () => void; }`.

Structure (three horizontal sections stacked vertically):

1. **Waveform area** (120px height, flex):
   - Use `useQuery({ queryKey: ["peaks", sample.jamId], queryFn: () => getPeaks(sample.jamId) })` to load peaks.
   - Render `<WaveformDetail audioUrl={audioUrl} peaks={peaks} onSeek={handleSeek} />` with NO bookmarks/sections/loops overlay props (preview shows single sample).
   - Audio URL: `http://localhost:23516/api/audio/${encodeURIComponent(sample.sourceJamName)}`
   - On mount (or when sample changes): call `useTransportStore.getState().loadJam(sample.jamId, sample.name, audioUrl, sample.durationSeconds)` and then `useTransportStore.getState().setActiveLoop({ startSeconds: sample.startSeconds, endSeconds: sample.endSeconds, label: sample.name })`.
   - The WaveformDetail component + transport store's activeLoop will constrain playback to the sample region (per A2 in RESEARCH.md).
   - While peaks are loading, show skeleton pulse animation (muted background, 120px height) per UI-SPEC.

2. **Metadata row** (below waveform, single horizontal row, md/16px padding):
   - Sample name (body/14px, font-semibold)
   - Key badge: `sample.keyDisplay ?? "--"`
   - BPM badge: `sample.tempoBpm ? Math.round(sample.tempoBpm) + " BPM" : "--"`
   - Duration: format as `m:ss` from `sample.durationSeconds`
   - TypeBadge for sample type
   - All in a horizontal flex row with gap-3.

3. **Action row** (below metadata, right-aligned, md/16px padding):
   - "Export Audio" button: accent filled (`bg-[#E8863A] text-white hover:bg-[#E8863A]/90`), primary CTA. `aria-label="Export audio for {sample.name}"`.
     - onClick: For bookmarks (`sample.sampleType === "bookmark"`), call `exportAudio(sample.id)` (existing command). For sections/loops, call `exportSampleAudio(sample.jamId, sample.startSeconds, sample.endSeconds, sample.name)` (new command per D-16).
     - Toast on success: `"Exported to {path}"` with "Show in Finder" action calling `revealInFinder(path)`.
     - Toast on error: `"Export failed"` with error description.
   - "Export Stems" button: ghost/secondary variant (`text-[#E8863A] hover:text-[#E8863A]/80`). `aria-label="Export stems for {sample.name}"`.
     - onClick: For bookmarks, call `separateStems(sample.id)`. For sections/loops, call `separateSampleStems(sample.jamId, sample.startSeconds, sample.endSeconds, sample.name)` (per D-17).
     - Toast: `"Separating stems for {sample.name}..."` then on success `"Stems ready for {sample.name}"`.
   - "Go to Jam" link button: ghost variant, accent text. `aria-label="Go to {sample.sourceJamName}"`.
     - onClick: `onNavigateToJam(sample.jamId)`.

Panel visibility: The panel renders based on whether `sample` prop is non-null. Use CSS transition on max-height or transform for the slide-up animation. When sample is null, panel height transitions to 0.

Escape key: `onClose` prop is called, which triggers `useSampleBrowserStore.clearSelection()` in the parent.

ARIA: `role="complementary"`, `aria-label="Sample preview"`. Export status announced via toast (already has aria-live built in from sonner).

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-09 | Tampering | Audio URL construction | accept | URL is constructed from `sample.sourceJamName` which comes from the trusted backend. No user input in URL path. localhost-only API. |
| T-07-10 | Tampering | Export from preview panel | mitigate | Export calls go through Tauri IPC to backend which uses `resolve_export_path()` for path sanitization (reuses T-07-02 mitigation). |
| T-07-11 | Denial of Service | Large peaks data for waveform | accept | Peaks are pre-computed and cached server-side. Loading peaks for a single jam is bounded by the jam's duration. react-query caches results. |
