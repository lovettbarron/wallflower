## Focal Point

**Primary visual anchor:** Bookmark regions on the waveform detail view. Colored translucent overlays sitting above the waveform content, visually layered above Phase 4 section markers. The waveform transforms from an analysis tool into an extraction workspace.

**Secondary anchor:** The stem mixer slide-up panel. A compact, focused audition interface that temporarily replaces the metadata area below the waveform. Four (or six) stem rows with solo/mute controls and a mini waveform per stem.

### Bookmark Creation Flow (D-01)

```
1. User click-drags on waveform detail view
2. Translucent selection region appears during drag (accent at 15% opacity)
3. On mouse-up, region snaps edges to nearby section/loop boundaries (D-02)
4. BookmarkPopover appears anchored to the region

+------------------------------+
| Name: [Bookmark 1         ]  |
|                               |
| Color: [*][*][*][*][*][*][*] |
|                               |
| Notes: [                   ] |
|        [                   ] |
|                               |
|     [Discard]  [Save Bookmark]|
+------------------------------+
```

- Selection region: accent color at 15% opacity during active drag. Cursor changes to `col-resize` on waveform.
- Snap-assist (D-02): When a drag edge is within 20px of a section/loop boundary, edge snaps to that boundary. A small tooltip appears: "Snapped to Verse A". Hold Option key to disable snap for free-form selection.
- BookmarkPopover: shadcn Popover, secondary background, md (16px) padding, 280px width.
- Name field: shadcn Input, auto-focused, default value "Bookmark {N}" (auto-incrementing). Selects all on focus for easy renaming.
- Color palette: 8 circular swatches (16px diameter), sm (8px) gap, inline row. Selected swatch shows 2px accent border ring. Default: next unused color in palette order.
- Notes field: shadcn Textarea, 2 rows, placeholder "Add notes...". Optional.
- Discard: ghost button. Save Bookmark: primary button (accent background).
- After save: region converts to a BookmarkRegion with the chosen color. Popover closes. Toast: "Bookmark saved".

### Bookmark Region on Waveform (D-04)

```
  |---- Loop A x4 ----|    (loop brackets, Phase 4)
  |  Intro  |  Verse A  |  (section markers, Phase 4)
  [===== bookmark region colored overlay =====]
  [=================== waveform ===================]
```

- Visual layer order (bottom to top): waveform fill, bookmark region fill (30% opacity), section marker lines (60% opacity), loop brackets, bookmark region border (2px, 80% opacity), bookmark label.
- Bookmark label: positioned at top-left inside the region. Label size (12px), bookmark color at full opacity. Truncated with ellipsis if region is too narrow. Hidden below 60px region width.

- Click on region: selects the bookmark (border becomes accent color), shows context menu trigger icon at top-right of region.
- Double-click on region: opens BookmarkPopover for editing.
- Right-click on region: opens BookmarkContextMenu.

```
|                                                    |
|  [==============>                    ] 62%         |
|  Chunk 7/12    ~18s remaining                      |
|                                                    |
|                         [Cancel Separation]        |
+--------------------------------------------------+
```

- Replaces the stem mixer content area during separation. Same panel dimensions.
- Progress bar: shadcn Progress, accent fill, full width minus md padding.
- Chunk info: label size (12px), muted-foreground. Format: "Chunk {N}/{total}" and "~{seconds}s remaining".
- Cancel Separation button: ghost button, destructive text. Cancels separation, closes panel, shows toast "Separation cancelled".

## Phase 5 Specific Notes

1. **Bookmarks are the core interaction.** Everything in this phase flows from bookmark creation. The drag-to-select gesture on the waveform must feel immediate and natural -- identical to selection behavior in Ableton/Logic. No mode switching required; just drag on the waveform.

2. **Snap-assist is helpful, not mandatory.** Section/loop boundaries from Phase 4 provide snap targets, but the user can always override with the Option key (D-02). The snap threshold of 20px provides guidance without frustration. If no analysis results exist, snap is simply not available and the interaction degrades gracefully to free-form selection.

3. **Stem mixer is temporary and focused.** The slide-up panel (D-10) replaces the metadata area, not the waveform. The waveform stays visible so the musician can see what they are listening to. The mixer is for quick audition and export -- not a production mixing environment. Dismiss returns to the full jam detail view.

4. **On-demand separation respects compute.** Separation only runs when "Export stems" is requested (D-09). Most bookmarks will use "Export audio" (simple time-slice). This means the separation progress UI only appears for the minority of exports, keeping the common path fast.

5. **Recording priority is maintained.** Separation pauses cleanly during recording (D-14) with clear status text. No error states, no data loss. The musician's active recording always takes precedence. This follows the same pattern established in Phase 3/4.

6. **Bookmark regions layer above section markers.** Visual hierarchy (D-04): waveform content at bottom, bookmark fill regions above, section marker lines above, loop brackets at top. Bookmarks represent user intent; sections represent AI analysis. Both are visible simultaneously but visually distinct through color palettes and rendering styles.

7. **Export folder structure mirrors Ableton browser expectations.** Folder hierarchy `exports/[Jam Name]/[Bookmark Name].wav` (D-06) maps directly to how Ableton's browser presents a folder tree. Musicians can point Ableton's browser at `~/wallflower/exports/` and drill into jams.

8. **Self-contained exports.** The JSON metadata sidecar (D-08) travels with the audio files. A collaborator receiving exported stems gets both the audio and the context (key, BPM, tags, source jam reference) without needing Wallflower installed.

9. **Cached stems avoid re-separation.** After separation completes, stems are cached on disk. Subsequent "Export stems" for the same bookmark opens the mixer immediately without re-running demucs. Cache invalidation occurs when the bookmark time range is edited.

10. **32-bit float downsampling is invisible.** Per EXP-06 and D-07, 32-bit float source recordings are automatically downsampled to the configured export bit depth (default 24-bit). No UI indicator or decision point -- it just works.
