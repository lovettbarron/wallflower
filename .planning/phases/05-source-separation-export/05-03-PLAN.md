       - `#[tauri::command] pub async fn export_audio(state: tauri::State<'_, AppState>, app: tauri::AppHandle, bookmark_id: String) -> Result<String, String>`:
         - Fetch bookmark by id from DB
         - Fetch jam by bookmark.jam_id from DB (need jam name, audio_path)
         - Get export settings from state (export_root, format, bit_depth)
         - Call resolve_export_path(export_root, jam_name, bookmark_name, format_extension)
         - Call export_time_slice(source_audio_path, dest_path, bookmark.start_seconds, bookmark.end_seconds, bit_depth)
         - Fetch analysis results for the jam (key, bpm, tags, etc.)
         - Call generate_sidecar with all metadata, write to {export_path_without_ext}.json
         - Record in exports table via create_export_record
         - Emit "export-complete" event with { bookmark_id, path }
         - Return the export path string

       - `#[tauri::command] pub async fn separate_stems(app: tauri::AppHandle, state: tauri::State<'_, AppState>, bookmark_id: String) -> Result<Vec<StemInfo>, String>`:
         - Fetch bookmark and jam from DB
         - Get separation settings (model_name, memory_limit)
         - Check stem_cache: if cached stems exist for this bookmark_id + model_name, return them immediately (skip separation)
         - If not cached:
           - Call SeparateStems gRPC RPC via the existing sidecar client
           - Stream progress: for each SeparationProgress message from gRPC stream, emit "separation-progress" Tauri event with { bookmark_id, status, current_chunk, total_chunks, percent_complete, estimated_seconds_remaining }
           - Between chunks, the Python side checks PriorityScheduler (or we check on Rust side and send pause/resume signals). Simpler: check scheduler.may_proceed() on Rust side before forwarding each chunk result, and if recording active, emit status "paused" and wait in a loop with sleep until may_proceed() returns true, then emit "resumed"
           - On completion, save each stem file info to stem_cache table
           - Return Vec<StemInfo> with stem_name and file_path for each stem

       - Define StemInfo struct:
         ```rust
         #[derive(Debug, Clone, Serialize, Deserialize)]
         pub struct StemInfo {
             pub stem_name: String,
             pub file_path: String,
             pub file_size_bytes: i64,
         }
         ```

       - `#[tauri::command] pub async fn export_stems(state: tauri::State<'_, AppState>, app: tauri::AppHandle, bookmark_id: String, stem_names: Vec<String>) -> Result<String, String>`:
         - Fetch bookmark, jam, and cached stems from DB
         - Get export settings
         - Resolve stems export dir via resolve_stems_dir
         - For each requested stem_name in stem_names, copy the cached stem WAV file to the export dir
         - Generate JSON sidecar with stems list
         - Record in exports table
         - Emit "export-complete" event
         - Return export directory path

       - `#[tauri::command] pub async fn cancel_separation(state: tauri::State<'_, AppState>) -> Result<(), String>`:
         - Set a cancellation flag that the separation stream loop checks
         - The gRPC stream will be dropped, signaling cancellation to Python side
