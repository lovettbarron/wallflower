    1. Create crates/wallflower-core/src/recording/writer.rs with CrashSafeWriter:
       - Struct fields: `writer: Arc<Mutex<Option<hound::WavWriter<std::io::BufWriter<std::fs::File>>>>>`, `flush_interval: Duration`, `file_path: PathBuf`, `flush_thread: Option<std::thread::JoinHandle<()>>`, `raw_fd: i32`
       - `new(path: &Path, spec: hound::WavSpec, flush_interval: Duration) -> Result<Self>`:
         * Open file with `std::fs::File::create(path)`
         * Store the raw fd via `use std::os::unix::io::AsRawFd; file.as_raw_fd()`
         * Wrap in `BufWriter::with_capacity(65536, file)` (64KB buffer for audio throughput)
         * Create `hound::WavWriter::new(buf_writer, spec)`
         * Wrap in `Arc<Mutex<Option<...>>>`
         * Start a flush timer thread that calls `self.do_flush()` every `flush_interval`
       - `write_samples(&self, samples: &[f32]) -> Result<()>`:
         * `try_lock()` on the mutex. If locked (flush in progress), buffer samples in a small `Vec<f32>` overflow buffer and write on next successful lock.
         * For each sample, call `writer.write_sample(sample)`
       - `do_flush(&self) -> Result<()>`:
         * Lock mutex, call `writer.flush()`, then `libc::fsync(self.raw_fd)` (per Research Pitfall 5)
       - `finalize(self) -> Result<PathBuf>`:
         * Take writer from mutex (`Option::take`), drop it (hound finalizes on drop), call `libc::fsync(raw_fd)`, return file_path
         * Signal flush thread to stop (set writer to None -- thread checks and breaks)
       - Unit tests:
         * `test_write_and_flush`: Create writer in temp dir, write 48000 samples (1 sec at 48kHz mono), flush, verify file is readable by `hound::WavReader` and has correct sample count
         * `test_finalize_produces_valid_wav`: Create writer, write samples, finalize, read back with hound, verify duration
         * `test_multichannel_write`: Create writer with 4-channel spec, write interleaved 4-channel data, verify spec.channels == 4 on readback (REC-09)

       - `new(config: RecordingConfig, event_tx: Sender<RecordingEvent>, scheduler: PriorityScheduler) -> Self`
       - `start(&self, storage_dir: &Path, jam_id: &str) -> Result<()>`:
         * Set scheduler recording = true (D-13, REC-08)
         * Get default input device via `device::get_default_input_device()`
         * Get supported config (prefer f32 format, use device's default sample rate)
         * Derive `hound::WavSpec` from cpal config: `WavSpec { channels: config.channels, sample_rate: config.sample_rate, bits_per_sample: 32, sample_format: SampleFormat::Float }`
         * Create output file path: `storage_dir.join(format!("{}.wav", jam_id))`
         * Create CrashSafeWriter with derived spec and config.flush_interval
         * Build cpal input stream with data callback that: (a) writes samples to CrashSafeWriter via try_lock, (b) computes RMS and sends LevelUpdate via event_tx.try_send, (c) runs silence detector and sends silence events
         * Build error callback that sends DeviceError event on StreamError::DeviceNotAvailable
         * Play the stream

       - `stop(&self) -> Result<PathBuf>`:
         * Pause/drop the cpal stream
         * Finalize the CrashSafeWriter (writes final header)

If device returns within timeout, rebuild stream into same writer (gap filled with silence based on wall-clock elapsed). If timeout expires, call stop().
