# Phase 3 — UI Design Contract

> Visual and interaction contract for recording engine UI: transport bar transformation, recording view, stop-recording confirmation, live waveform, menubar status, and global hotkey feedback. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (customized preset, inherited from Phase 2) |
| Preset | custom -- dark base, warm accent tokens, extended border-radius |
| Component library | radix (via shadcn) + custom Tailwind components |
| Icon library | lucide-react |
| Font | Plus Jakarta Sans (400, 600 weights via @fontsource) |

**Phase 3 continuity:** No design system changes from Phase 2. All tokens, typography, and color palette are inherited. Phase 3 adds a recording-specific color (recording red) and new component states to the existing system.

## Color

### Inherited Palette (unchanged)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(220 16% 10%) / #151921 | Page background |
| Secondary (30%) | hsl(220 14% 14%) / #1D2129 | Cards, transport bar background, metadata panels |
| Accent (10%) | hsl(28 90% 58%) / #E8863A | Per Phase 2 reserved list (play button, waveform, tabs, CTAs, focus ring) |
| Destructive | hsl(0 72% 51%) / #D93636 | Destructive actions only |

### Phase 3 Addition: Recording Red

| Role | Value | Usage |
|------|-------|-------|
| Recording active | hsl(0 85% 55%) / #E53E3E | Recording indicator dot, record button fill, transport bar left border when recording, live waveform fill |
| Recording active dimmed | hsl(0 60% 35%) / #8F2A2A | Recording waveform already-captured region (same relationship as waveform-played to waveform-primary) |

**Why a separate red from destructive:** Recording red (hsl 0 85% 55%) is brighter and warmer than destructive red (hsl 0 72% 51%). Recording is not a destructive action -- it is the primary creative action. The brighter red ensures the recording state is unmistakably visible at a glance, even in peripheral vision. Destructive red remains reserved for delete/remove actions.

### Accent Reserved For (Phase 3 update)

Phase 2 accent list remains unchanged. Recording elements use Recording Red, NOT accent:

1. Record button in transport bar (recording-red filled circle)
2. Live waveform fill color during recording (recording-red)
3. Recording indicator dot (pulsing recording-red)
4. Transport bar left border during recording (3px solid recording-red)

Accent (amber) continues to be used for:
- Playback controls (disabled during recording per D-04, but styled consistently)
- Waveform of completed recordings (viewed after recording stops)
- Active tab indicator, focus rings, CTAs

## Focal Point

**Primary visual anchor during recording:** The transport bar transforms with a 3px recording-red left border, pulsing record indicator, and elapsed time -- always visible at the bottom of the screen. The recording state is unmissable.

**Secondary anchor:** The live waveform growing in recording-red in the jam detail view. The waveform builds left-to-right as audio is captured, providing continuous visual feedback that recording is working.

## Layout Contract

### App Shell During Recording (D-11: locked to recording view)

```
+--------------------------------------------------+
| [Wallflower]              (native title bar)      |
+--------------------------------------------------+
| [ Library ]  [ Settings ]  (tabs -- DISABLED)     |
+--------------------------------------------------+
|                                                    |
|  [<- Library]   (DISABLED, greyed out)             |
|  Recording: jam-2026-04-19-session                 |
|                                                    |
|  [====== live waveform (growing) ======]  200px    |
|                                                    |
|  --- Metadata ---                                  |
|  (full MetadataEditor, identical to Phase 2)       |
|                                                    |
+--------------------------------------------------+
| * REC  0:14:32   Zoom F3 (Stereo)  [||||] [Stop] |
+--------------------------------------------------+
```

- Tab bar: Visually present but non-interactive during recording. Tab text uses muted-foreground color. Cursor: not-allowed. Prevents navigation away from recording (D-11).
- Back navigation: Greyed out (muted-foreground), non-interactive. Tooltip on hover: "Stop recording to return to library."
- Recording title: Heading size (20px/600), with a pulsing recording-red dot (8px circle) to the left. Dot pulses with a CSS animation: opacity 1.0 to 0.3 over 1.5 seconds, infinite loop.
- Live waveform: 200px height (same as Phase 2 detail waveform). Recording-red fill. Grows left-to-right. Overview waveform (48px) also shows the growing recording.
- Metadata section: Identical to Phase 2 jam detail. All fields are editable during recording (D-10, REC-05). Live-save pattern unchanged.
- Bottom padding: 56px to clear transport bar (unchanged).

### Transport Bar -- Playback Mode (unchanged from Phase 2)

```
+--------------------------------------------------+
| [<<] [|>] [>>]  0:00 / 1:23:45   jam-name  [*]  |
+--------------------------------------------------+
```

- Record button (*) added to the right side of the transport bar. 36px circle, transparent background with recording-red border (2px). Lucide `Circle` icon (filled) in recording-red, 14px.

### Transport Bar -- Recording Mode (D-01)

```
+--------------------------------------------------+
|  * REC  0:14:32   Zoom F3 (Stereo)  [||||] [Stop]|
+--------------------------------------------------+
```

- 3px recording-red left border on the entire transport bar (replaces standard left edge).
- Recording indicator: 8px pulsing recording-red dot (same animation as title dot).
- "REC" label: Label size (12px), recording-red color, uppercase, font-weight 600.
- Elapsed time: Label size (12px), foreground color, tabular-nums. Format: "H:MM:SS" (no leading zero on hours, e.g., "0:14:32" or "1:02:15").
- Device name: Body size (14px), muted-foreground color. Shows active input device + channel config. Truncated with ellipsis if too long.
- Input level meter: 4px height, 120px width. Background: border color (#323844). Fill: gradient from recording-red (left, hot) to accent amber (right, peak). Updates at ~15fps from backend level data. Shows current input level as a horizontal bar.
- Stop button: Secondary button style (surface-elevated background, foreground text). Label: "Stop". 8px border-radius. On click: opens StopRecordingDialog (D-12).
- Playback controls (play/pause, skip) are hidden during recording (D-04).

### Transport Bar -- Device Disconnected During Recording (D-05)

```
+--------------------------------------------------+
|  * REC  0:14:32   ! Reconnect device   [||||] [Stop]|
+--------------------------------------------------+
```

- Device name area shows warning: "! Reconnect device" in destructive color with AlertTriangle icon (lucide, 14px).
- Recording indicator dot switches from pulsing to solid recording-red (no animation -- recording is paused/waiting).
- Input level meter shows empty (no fill).
- Transport bar left border switches from recording-red to destructive red.
- If device reconnects: reverts to normal recording transport state. Toast: "Recording resumed -- {device name} reconnected."

### Stop Recording Dialog (D-12)

```
+--------------------------------------+
|  Stop Recording?                      |
|                                       |
|  1 hour, 14 minutes captured.         |
|                                       |
|           [Cancel]    [Stop Recording] |
+--------------------------------------+
```

- Dialog uses shadcn Dialog component with Phase 2 styling.
- Background overlay: hsl(220 16% 10%) at 80% opacity.
- Dialog surface: Secondary background (#1D2129), 12px border-radius, border color (#323844) 1px border.
- Title: Heading size (20px/600), foreground color.
- Body: Body size (14px/400), muted-foreground color. Duration formatted in human-readable words (e.g., "1 hour, 14 minutes" not "1:14:32").
- Cancel button: Secondary style (surface-elevated background, foreground text, 8px border-radius).
- Stop Recording button: Destructive style (destructive background #D93636, white text, 8px border-radius). This IS a destructive action (ending an irreversible recording session).
- Focus: Stop Recording button is NOT auto-focused. Cancel is auto-focused to prevent accidental recording termination. Tab order: Cancel -> Stop Recording.
- Keyboard: Escape dismisses dialog (same as Cancel). Enter activates focused button.

### Silence Markers on Waveform (D-03)

Silence markers appear as semi-transparent overlays on the waveform:

- Visual: Vertical region with background hsl(220 14% 14%) at 40% opacity over the waveform area. Top and bottom edges marked with 2px horizontal lines in muted-foreground color.
- Timestamp labels: Label size (12px), muted-foreground, positioned above the waveform at the start and end of each silent region.
- Configurable threshold: Set in Settings page (existing settings infrastructure from Phase 1). Default: -40dB. Range: -60dB to -20dB.

## Interaction States

### Record Button (in playback-mode transport bar)

| State | Visual |
|-------|--------|
| Default | 36px circle, transparent background, 2px recording-red border, filled circle icon (14px) in recording-red |
| Hover | Background fills with recording-red at 15% opacity |
| Active (pressed) | Background fills with recording-red at 25% opacity |
| Recording active | Button is replaced by full recording transport layout (see Recording Mode above) |
| Disabled (no input device) | Border and icon in muted-foreground. Tooltip: "No audio input device detected" |

### Recording State Transition

| Transition | Animation |
|------------|-----------|
| Start recording | Transport bar slides content change over 200ms ease-out. Playback controls fade out, recording controls fade in. Left border appears with 150ms ease-in. |
| Stop recording | Reverse of start. Recording controls fade out, playback controls fade in. Left border fades over 150ms. |
| New jam created | View navigates to jam detail automatically. Waveform area shows "Preparing to record..." placeholder text (muted-foreground, centered) for up to 500ms while the backend initializes. |

### Live Waveform

| State | Visual |
|-------|--------|
| Initializing | Waveform area shows empty secondary background. Centered text: "Preparing..." in muted-foreground, label size. |
| Recording | Waveform grows left-to-right in recording-red. New audio data appends to the right edge. View auto-scrolls to keep the right edge visible. |
| Silence detected | Waveform amplitude drops to near-zero. Silence marker overlay appears in real time once the silence duration exceeds the threshold. |
| Device disconnected | Waveform stops growing. A gap region appears (dashed vertical line in destructive color, 2px). Text above gap: "Device disconnected" in label size, destructive color. |
| Device reconnected | Waveform resumes growing after the gap. Toast notification confirms reconnection. |

### Navigation Lock (D-11)

| Element | Locked State |
|---------|-------------|
| Tab bar (Library / Settings) | Text color: muted-foreground. Cursor: not-allowed. Click: no action. No tooltip needed -- the recording state is visually obvious. |
| Back navigation ("< Library") | Text color: muted-foreground. Cursor: not-allowed. Click: no action. Tooltip on hover: "Stop recording to return to library." |
| Jam cards (if somehow visible) | Not applicable -- user cannot reach timeline view during recording. |

### Metadata Editing During Recording

Identical to Phase 2 metadata editing. All fields (tags, collaborators, instruments, notes, location, photos) are fully editable. Live-save operates normally. No changes to the interaction model.

### System Tray Icon

| State | Icon | Menu Items |
|-------|------|------------|
| Idle (not recording) | Wallflower app icon (monochrome, 16x16 template image for macOS menu bar) | "Start Recording" / "Open Wallflower" / separator / "Quit Wallflower" |
| Recording | Recording variant icon (red dot overlay or alternate icon indicating active recording) | "Recording -- {elapsed time}" (disabled, informational) / "Stop Recording" / "Open Wallflower" / separator / "Quit Wallflower" |

- Tray icon is a native macOS template image (single-color, system-tinted).
- Recording state: either a distinct icon variant or a red dot badge. Tauri system tray API determines the exact mechanism.
- Menu is a native NSMenu (not a web popover). Items use system font at system size.
- "Stop Recording" in the tray menu triggers the same StopRecordingDialog in the main window (brings window to front first).

### Global Hotkey Feedback (D-08, D-09)

| Action | Feedback |
|--------|----------|
| Cmd+Shift+R pressed (start recording) | If app is focused: recording starts immediately with transport bar transition. If app is not focused: recording starts, tray icon changes to recording state, native macOS notification: "Recording Started -- Wallflower is now recording from {device name}." |
| Cmd+Shift+R pressed (stop recording) | StopRecordingDialog opens. If app is not focused: window is brought to front, then dialog appears. Recording does NOT stop without confirmation (D-12). |
| Hotkey pressed with no input device | Native notification: "Cannot Record -- No audio input device detected." No state change. |

## Phase 3 Specific Notes

1. **Recording red is NOT accent.** The recording state uses a dedicated recording-red color (hsl 0 85% 55%) that is distinct from both the accent amber and the destructive red. This three-color separation ensures: amber = creative playback, red = active recording, dark red = destructive actions. Each has a distinct meaning.

2. **Transport bar is the single source of recording truth.** The transport bar always reflects the current recording state. A musician glancing at the bottom of the screen can instantly see: recording/not recording, elapsed time, device status, input level. No need to look elsewhere.

3. **Navigation lock is simple, not clever.** During recording, tabs and back-nav are greyed out. No modal overlays, no warnings -- just visually disabled. The recording state in the transport bar makes it obvious why navigation is locked. The musician's focus stays on the metadata and live waveform.

4. **Stop-recording confirmation defaults to Cancel.** This is deliberate (D-12). Accidentally stopping a recording is worse than accidentally dismissing the dialog. Cancel is auto-focused so Enter key dismisses without stopping. The musician must deliberately Tab to and activate "Stop Recording."

5. **System tray is native, not web.** The tray icon, menu, and interactions are all handled by Tauri's system tray API on the Rust side. The UI-SPEC documents what the user sees and the copy they read, but the implementation is not React components.

6. **Live waveform is a new component.** Unlike Phase 2's wavesurfer.js-based static waveform, the recording waveform receives real-time audio data and renders progressively. This may use wavesurfer.js with a custom renderer or a canvas-based approach -- the visual contract (color, size, behavior) is specified here; implementation approach is the planner's decision.

7. **Input level meter is intentionally minimal.** A 4px horizontal bar in the transport bar provides just enough feedback to confirm audio signal is present. This is not a professional metering tool -- it is a confidence indicator for musicians who want to know "is it picking up sound?"
