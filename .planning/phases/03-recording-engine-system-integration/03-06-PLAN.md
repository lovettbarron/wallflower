  <action>
  Two changes in src/components/tauri-event-listener.tsx:

  **Change 1 — recording-started handler (currently line 214-219):**
  The current handler only shows a toast. Replace it so it ALSO sets the zustand recording store state. The `RecordingStartedPayload` already includes `jamId`, `jamName`, and `deviceName`.

  Replace the recording-started listener callback (the function inside `listen<RecordingStartedPayload>("recording-started", ...)`) with:
  ```typescript
  (event) => {
    const { jamId, deviceName } = event.payload;
    // Set recording state in zustand store — this triggers:
    // 1. page.tsx renders RecordingView (isRecording === true)
    // 2. TransportBar switches to recording mode
    // 3. Navigation locks to recording view
    // 4. Elapsed timer useEffect starts (isRecording dependency)
    useRecordingStore.setState({
      isRecording: true,
      recordingJamId: jamId,
      deviceName: deviceName,
      elapsedSeconds: 0,
      rmsDb: -100,
      deviceDisconnected: false,
      silenceRegions: [],
      showStopDialog: false,
      levelHistory: [],
    });
    toast(`Recording from ${deviceName}`, { duration: 3000 });
  }
  ```

  Use `useRecordingStore.setState()` (the static method on the zustand store) rather than individual setter actions. This is a single atomic state update that mirrors what `startRec()` does after the Tauri command resolves (see recording.ts line 42-50), but without calling `startRecording()` again (the backend already started recording — this event IS the confirmation).

  **Change 2 — recording-state-changed handler (currently line 122-149):**
  The `state === "recording"` branch currently only handles reconnection (checks `wasDisconnectedRef.current`). Add a fallback: if `state === "recording"` and the store does NOT have `isRecording === true`, set it. This handles edge cases where `recording-state-changed` fires before or instead of `recording-started`.

  Update the `else if (state === "recording" && wasDisconnectedRef.current)` branch to:
  ```typescript
  } else if (state === "recording") {
    if (wasDisconnectedRef.current) {
      // Device reconnection case
      setDeviceDisconnected(false);
      wasDisconnectedRef.current = false;
      toast.dismiss("device-disconnect");
      toast.success("Recording resumed", { duration: 3000 });
    }
    // Ensure store reflects recording state (covers edge cases
    // where state-changed fires but recording-started was missed)
    const currentState = useRecordingStore.getState();
    if (!currentState.isRecording) {
      useRecordingStore.setState({ isRecording: true });
    }
  }
  ```

  This removes the `wasDisconnectedRef.current` requirement from the outer condition — the reconnection toast logic is preserved inside the inner `if`, but the `isRecording: true` safety net runs unconditionally when state is "recording".
