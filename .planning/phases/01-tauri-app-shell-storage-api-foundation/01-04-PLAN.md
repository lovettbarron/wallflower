    - Test 2: Sample values are correctly clamped and scaled. Input 1.0f32 -> 8388607i32 (2^23-1), input -1.0f32 -> -8388608i32 (-2^23), input 0.0f32 -> 0i32.

  <action>
    Create a new `audio` module in wallflower-core (per STOR-05 gap closure):

    1. Create `crates/wallflower-core/src/audio/mod.rs` with `pub mod downsample;`

    2. Create `crates/wallflower-core/src/audio/downsample.rs` with:
       - `pub fn downsample_32f_to_24i(source: &Path, dest: &Path) -> Result<(), DownsampleError>` 
       - Uses hound::WavReader to open source, verify it is 32-bit float (spec.sample_format == SampleFormat::Float && spec.bits_per_sample == 32)
       - Creates hound::WavWriter with output spec: same channels, same sample_rate, bits_per_sample=24, SampleFormat::Int
       - Reads samples as f32, converts each: clamp to [-1.0, 1.0], multiply by 8388607.0 (2^23 - 1), round to i32
       - For values at exactly -1.0, use -8388608 (full negative range of 24-bit)
       - Write each converted sample via writer.write_sample(value_i32)
       - Finalize writer
       - Define `DownsampleError` enum with variants: IoError, NotWav, NotFloat32, HoundError

    3. Add `pub mod audio;` to `crates/wallflower-core/src/lib.rs`

    Conversion formula per sample:
    ```
    let clamped = sample.clamp(-1.0, 1.0);
    let scaled = if clamped >= 0.0 {
        (clamped * 8388607.0) as i32
    } else {
        (clamped * 8388608.0) as i32
    };
    ```

    Tests go in `downsample.rs` in a `#[cfg(test)] mod tests` block. Use tempfile for input/output WAV files. Create test WAVs with hound (same pattern as metadata.rs tests).
